## 0. 一句話定位

把「**審查還是抗審查**」這個兩難,做成一個**你能親手操作、且每句話都有出處**的互動科普網站;以 Matters 自身 **2018–2025 的治理史**與 **2025 春節 DNS-RPZ 誤封事件**為主線,對三種受眾(一般人 / 政策關係人 / 開發者)說清楚:垃圾偵測、排序演算法、抗審查,到底是什麼、為什麼難。

**三個北極星**
1. **互動即論證**(form embodies content):拉動分數線就會親手誤傷無辜,而不是用文字說「會誤傷」。
2. **完全引用**:每個主張都連到簡報 / 逐字稿 / GitHub commit / 報告 / 個人文章。
3. **網站自身就是證明**:部署在 IPFS,網域被封也讀得到——用結果示範抗審查(呼應 DNS-RPZ 事件)。

## 1. 敘事主軸(鎖定建議)

提案書的真正命題是「**透明開盒(Algorithmic Transparency)的兩難**」[原始提案內容]。核心張力一句話講完(引自模型更新報告):

> 「垃圾抑制與異議消音,在技術上共用同一機制——一個分數加一個門檻;在價值上卻彼此對立。**在不確定時,寧可漏看一篇垃圾,不願消音一篇表達。**」[MODEL_UPDATE_REPORT.md]

整體處置上限的設計原則(全站可反覆強調的金句):

> 「**不刪除,只是不再被看見**」——上限是降低可見度而非刪除;不確定的交給人;一切可回復、可申訴、可稽核。[PROGRESS_REPORT.md]

**結構 = 三支柱 × 時間軸 × DNS-RPZ 收束**
- **三支柱**(三個主章):`審查` ｜ `抗審查` ｜ `首頁排序`。
- **時間軸當脊椎**:每柱內部用真實的「治理事件 → 服務升級」推進(素材現成,見 §3)。
- **DNS-RPZ 事件當高潮**:它同時牽動三柱(垃圾→打詐掃描→網域封鎖→IPFS 續存),放在收束,把全站綁成一個故事。

> 這同時實現你最初列的三條軸線(升級類別 / 時間軸 / 內容類別),但用**一條脊椎**收編,讀者不會迷路。

## 3. 內容大綱(逐章 × 素材對應 × 招牌互動)

> 每章標「敏感度」:🟢可公開 / 🟡需你確認可揭露程度 / 🔴僅概念不揭參數(提案明訂:不揭可被繞過的細節)。

### 序章 — 被治理者,同時是治理者 🟢
- 鉤子:用一個具體事件開場(建議 **2019 反送中假帳號**:仿冒江澤民/吳敦義,吳敦義辦公室真的來信)→ 點出「中小平台被迫長出治理」。[治理史時間線]
- 數據錨:台灣 73% 用社群、50% 不信任隱私保護、79% 認為充斥假訊息(TWNIC《2024 台灣網路報告》)→「必要卻不被信任」。[原始提案內容]

### 第二部 ── 抗審查:為什麼也必須抗、怎麼抗 🟢
- 2.1 中心化的問題:劍橋分析、土耳其封維基;廣告經濟扭曲內容市場。[去中心內容發佈.md]
- 2.2 **內容定址 vs 位置定址**:HTTP 綁網域/IP,刪檔或改 DNS 就失效;IPFS 用內容指紋,只要有節點持有就抗審查。IPFS(2018.10)→ IPNS 固定地址(2022.09)→ ENS 好記名(2022.12)。開源工具 `ipns-site-generator`。[去中心內容發佈.md]
- 2.3 **分層治理**:平台可下架,使用者仍持有去中心網址 → 平衡平台管理與個人權益。這是提案的核心倡議。[原始提案內容]
- 互動:`location vs content addressing` 對照 demo(刪掉中心伺服器,IPFS 版本仍在)。

### 第三部 ── 首頁排序:大家口中的「演算法」🟡
- 3.1 **降權 vs 刪除**:垃圾是被二元過濾(不出現在公共頁面),不是降權;熱門排序另有公式。[platform-governance §03]
- 3.2 **新首頁(2025.07)** 的排序邏輯與權重取捨。[matters-moderation-ranking-report-20260530.md]
- 互動:**排序 split-screen**(改權重,兩欄首頁同時重排;The Markup「Split Screen」式)。

### 第四部 ── 管理員手段(七種處置)🟢
- 七種處置:**註銷 / 凍結 / 禁言 / 折疊 / 小黑屋 / 降噪 / 打掃**,嚴厲度與可逆性光譜。[platform-governance §02]
- 小黑屋(2023.04.13 後臺版上線)的設計動機:本來靠「註銷」防洗版視覺干擾,有了小黑屋就「踢進去就好,很少註銷了」。[治理史時間線、內部訪談:小黑屋逐字稿]
- 互動:處置光譜尺(嚴厲度 × 可逆性)。

### 第五部 ── 群眾協力 🟢
- **守望相助隊**:可信市民移除留言廣告、**每個動作公開可稽核**(誰/何時/為何/申訴狀態)、3 檢舉自動折疊、申訴 `hi@matters.town`、**v1 階段 AI 不得自動移除**。[community-watch]
- **救生員 / 海巡 bot**:Tier-1 檢舉(≥0.55)/ Tier-2 移除(≥0.80,預設關閉);ring gate 覆寫分數;護欄(dry-run、kill-switch、**反轉率 brake**:近期被推翻率 >10% 就罷工)。[coastguard-bot]
- 歷史對照:社區訴訟制度(2019.08)→ 建築師陪審團(2021.03)的群眾治理嘗試與失敗教訓(抱團投票、小號拉票)。[治理史時間線]
- 互動:海巡 bot 三階段護欄模擬。

### 收束 ── 2025 春節 DNS-RPZ 誤封事件 🟡(政治敏感,謹慎)
- 敘事鏈:**二次詐騙文(2022.06 起)→ 觸發台灣政府打詐掃描 → TWNIC 以 DNS-RPZ 改寫解析、封鎖部分台灣用戶連線(1/24 傍晚起)→ 適逢連假,程序救濟未能即時啟動 → IPFS 讓內容仍可經指紋讀取**。[網域封鎖事件公告、原始提案內容、去中心內容發佈.md]
- 建言(建設性、基於事實、不指控):跨層級治理、跨機關通報流程、證據格式標準的制度缺口。提案明訂**只以平台自身困境為案例、產出倡議方向,不涉入既有國家政策流程**。[原始提案內容]
- 這是全站「抗審查為何必要」最有力的真實論據,也是 form-embodies-content 的終點(此站本身放 IPFS)。

### 延伸區(加分,非核心)
- **NCC 合規對照**:用《透明度報告指引》《申訴及救濟機制指引》對照 Matters 現況,做一個合規自評。[NCC合規實驗/*.pdf]

## 9. 工作流程與分工(沿用你的工具規劃,微調)
1. **deep-research**:只補**外部**脈絡(他國平台治理、DSA、CIB、DNS-RPZ 國際案例),**不**研究你手上的一手料。
2. **doc-coauthoring**:一手料直接寫成有引用的科普稿;敘事段用 中性的對外科普語氣(非個人語氣);豆泥既有文章當內容素材納入,技術段保持精確中性。
3. **web-artifacts-builder**:每個招牌互動先做原型驗證「學得會嗎」→ 找 3–5 位非技術朋友實測。
4. 驗證過的元件移植進 Astro。
5. **canvas/brand**:插畫用固定 prompt + style-ref 批量產;抽 brand skill 鎖風格(做過 2–3 次再抽,別過早)。
6. **skill-creator**:重複工(引註元件、事件卡片、圖風格)穩定後抽成 skill。
