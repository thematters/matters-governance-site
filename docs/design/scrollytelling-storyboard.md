# 主敘事 Scrollytelling 腳本：綠 vs 紅

> 這是全站的**主軸**：一條單頁、向下捲動的敘事，用「綠色＝正常／紅色＝垃圾」的方塊，把審查的意義一次走完。深度文本（14 章報告）作為「深入閱讀 →」外連。
> 來源：使用者 2026-06-19 口述的真實需求與分鏡。

## 0. 名詞與技術

- 格式：**scrollytelling（捲動敘事）**；圖固定、隨捲動更新的段落＝**sticky scrollytelling**。
- 技術：**IntersectionObserver / Scrollama** 觸發，內容方塊用 CSS transform + Canvas/SVG。落在可封存、no-JS 可降級為靜態圖文的選擇內。
- 核心視覺語言：**內容方塊（card）**。綠＝正常用戶內容、紅＝垃圾內容；灰＝被誤殺（偽陽性）；紫＝可互動信號。

## 1. 架構決定

**主頁 = 這條 scrollytelling 主敘事**（高互動閱讀體驗）。每一幕的延伸知識用「深入閱讀 →」外連到對應的報告章節分頁。也就是：

```
主頁（scrollytelling 主軸：綠紅 feed 故事 + 偽陽性滑桿）
 ├─「深入閱讀」→ /偵測模型（報告 03）
 ├─「深入閱讀」→ /偽陽性（報告 05）
 ├─「深入閱讀」→ /集團掃描（報告 04）
 ├─「深入閱讀」→ /處置・小黑屋（報告 06）
 ├─「深入閱讀」→ /群眾協力（報告 08）
 ├─「深入閱讀」→ /首頁排序（報告 07）
 └─「深入閱讀」→ /抗審查（報告 09）、/DNS-RPZ（報告 10）…
```

與「可探索插畫 hero」的關係：**插畫剖面世界當開場簾幕**（Scene 0），捲動後溶接進綠紅 feed 的 scrollytelling。插畫是氛圍與入口，card 敘事是主要機制。

## 2. 分鏡（逐幕）

> 每幕標：①畫面與動畫 ②教學點 ③互動 ④對應章節 ⑤資料需求

### Scene 0 — 簾幕：可探索插畫世界
- ①：波隆納剖面世界 hero（產生的圖片 1）淡入；標題「審查，還是抗審查？」。輕微視差。
- ②：定場——這是一個必須一邊抗審查、一邊清垃圾的平台。
- ③：向下捲動的提示（紫色脈動）。

### Scene 1 — 沒有審查的首頁
- ①：畫面溶接成一個「首頁 feed」。方塊開始從下往上流入：先是幾張綠色（正常用戶內容）。
- ②：「假設完全沒有審查，你看得到所有內容。」
- ④：對應「什麼是首頁／公共頁面」。

### Scene 2 — 紅色淹沒
- ①：隨捲動，紅色方塊越來越多、越流越快，逐漸蓋過綠色（呼應真實：每 5 篇有 4 篇是垃圾）。feed 變得無法閱讀。
- ②：「要維持一個公共首頁，就得把紅色排除掉——所以需要某種演算法／模型去過濾。」
- ⑤：紅色 card 的真實垃圾類型（去識別 mockup）：色情廣告、賣藥、二次詐騙、SEO 農場、政治洗版。

### Scene 3 — 模型怎麼學會分辨
- ①：每張 card 浮現一個「分數 0–100」。鏡頭拉到「訓練」：一疊真實案例（綠/紅 mockup）被「人工標記」貼上標籤、餵給模型；模型學會給分。
- ②：簡單講模型怎麼訓練——用大量人工標記的真實正常/垃圾案例去學分辨。
- ④：深入閱讀 → /偵測模型（03）。
- ⑤：~6–10 對「明顯正常 vs 明顯垃圾」的去識別案例。

### Scene 4 — 過濾：feed 變乾淨
- ①：模型掃過，紅色 card 一張張被抽離（滑走/變暗），綠色留下，feed 恢復可讀。
- ②：「分數過線就排除——首頁清爽了。」
- ④：降權 vs 刪除（垃圾是二元剔除，不是排後面）。

### Scene 5 — 偽陽性：招牌互動（滑桿）★
- ①：sticky 場景。一條**分數門檻線**橫亙 feed；上方一個滑桿。畫面同時顯示「被擋的垃圾」與「被誤殺的正常」。
- ③ **互動**：使用者拖動滑桿調整門檻——
  - 門檻往下（嚴）：紅色幾乎全擋下，但**綠色開始被誤殺**（變灰、掉落）= 偽陽性。
  - 門檻往上（鬆）：誤殺少了，但紅色漏回來 = 偽陰性。
  - 即時數字：擋下垃圾數、誤殺正常數。
- ②：「偽陽性＝誤殺正常內容。為什麼會有偽陽性？因為模型不確定、內容有灰色地帶。目標是讓偽陽性越低越好——**寧可漏看一篇垃圾，不願消音一篇表達**。」
- ④：深入閱讀 → /偽陽性（05）。
- ⑤：幾張「邊界案例」去識別 mockup——看起來像垃圾、其實是正常（政治評論、學術考據、詩、防詐科普），這些最容易被誤殺。

### Scene 6 — 模型之外的工具箱
- ①：偽陽性結束後，敘事繼續。依序用小動畫帶出：
  - **打掃 + 小黑屋**：紅 card 不是被刪除，而是被移出公共視野（「不刪除，只是不再被看見」）。
  - **人工標記**：被誤殺的綠 card 被人撿回、重新標記 → 回灌模型（人在迴圈裡）。
  - **新型態模型訓練**：模型會「自我中毒/錯殺放大」（可樂倒進麵團），用大模型重新校正。
  - **群眾智慧 / 守望相助隊**：一群人（產生的圖片 6 提燈場景）一起把紅 card 收進籃子；公開可稽核。
- ④：深入閱讀 → /處置（06）、/群眾協力（08）。

### Scene 7 — Ring：集團偵測
- ①：一張紅 card 分數很低（0.09）幾乎漏網；但鏡頭拉遠，發現它是一**整群幾乎一樣的 card、來自很多帳號**，彼此用線連成一張網 → 整網一起被框住。
- ②：「單篇模型抓不到的，集團偵測抓得到——因為它**抓行為、不抓字句**：同一模板、大量帳號、短時間。這跟剛剛的審查模型不一樣。」
- ④：深入閱讀 → /集團掃描（04）。

### Scene 8 — 一輪走完：審查的意義
- ①：feed 回到綠色為主、乾淨可讀；回顧剛剛的鏈條（過濾→偽陽性→工具箱→Ring）。
- ②：「這就是審查在做的事，以及它為什麼難。」

### Scene 9 — 更大的問題（收尾 + 外連）
- ①：畫面拉高，露出底層。三個 teaser 卡：
  - **抗審查**：平台能擋紅色，也可能擋掉它不該擋的 → IPFS 讓內容被刪也還在。
  - **末日爆滑（Doomscrolling）+ 首頁排序**：讓你黏在平台、無止盡下滑的演算法——「決定誰被看見」才是真正的權力。
  - **DNS-RPZ**：當整個網域被封鎖時。
- ②：「審查與抗審查、被看見與被淹沒，是同一件事的兩面。」
- ③：每個 teaser →「深入閱讀」外連到 09 / 07 / 10。

## 3. 招牌互動規格：偽陽性分數線（綠紅 feed 版）

- 一組固定的 card（綠 N 張、紅 M 張，每張帶一個「真實分數」）。
- 滑桿 = 門檻 t（0–100）。card.score ≥ t → 判為垃圾（移除）。
- 視覺：紅且被擋＝正確；紅但漏回＝偽陰；綠但被擋＝**偽陽性（灰掉/掉落，最該避免）**；綠且留＝正確。
- 即時計量：擋下垃圾、漏網垃圾、**誤殺正常**。
- 教學引導：拖到最嚴 → 看誤殺暴增；拖到最鬆 → 看垃圾淹回。甜蜜點在中間。
- 範本參考：Parable of the Polygons（拖一下就親手造成後果）、R2D3（資料點落在移動分界線兩側）。

## 4. 我需要你提供的唯一一塊料：去識別案例卡

scrollytelling 的綠/紅 card 要從**真實 Matters 案例**來（但去識別，用 mockup 圖示或改寫內文，讓一般人一眼看出正常 vs 垃圾）。請給我三組：

1. **明顯正常（綠）**：~8 張，各類型（生活、書評、藝文、時事評論…）。
2. **明顯垃圾（紅）**：~8 張，各類型（色情廣告、賣藥、二次詐騙、SEO 農場、政治洗版）。
3. **邊界／易誤殺（綠但像紅）**：~6 張（政治評論、學術考據、詩、防詐科普——這些是偽陽性的主角）。

每張只需：一句改寫後的內文（或標題）＋類型標籤＋（可選）一個 mockup 分數。**不需要真實原文**，去識別即可。沒有的話我先用合成範例頂著，你之後替換。

## 5. 技術實作備註

- Astro 頁，scrollytelling 用一個 vanilla island：IntersectionObserver 觸發各 scene 狀態；card 用絕對定位 + CSS transform 動畫；偽陽性互動用 Canvas 或 DOM。
- **無障礙/可封存**：每幕的結論用文字寫出（screen reader / no-JS 可讀）；尊重 `prefers-reduced-motion`（關閉飛入動畫、改靜態）；滑桿有鍵盤操作與 ARIA。
- 不劫持原生捲動；只用捲動「觸發」動畫，使用者隨時可正常上下滑。
